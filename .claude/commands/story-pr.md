# /story-pr — Per-Story Draft PR + Block-Until-Merged

Read-only with respect to LaiM artifacts. Touches git and GitHub (via `gh`). No state.json writes, no SKILL edits.

This command is an optional, user-invoked add-on. Run it only after `/implement` has reported a Gate 5 pass for the current story.

## 1. Read current story

Read `docs/state.json` with the Read tool. Extract:

- `currentStory.storyId`
- `currentStory.slug`
- `currentStory.title`
- `feature` (top-level field) — e.g. `jupiter-swap-template`

If `currentStory` is missing or null:
```
No story context in docs/state.json. Nothing to PR.
```
→ HALT.

Compute:
```
STORY_BRANCH = story/{storyId}-{slug}
```

## 2. Sanity checks

### 2a. Branch matches

```bash
git branch --show-current
```
Must equal `STORY_BRANCH`. If not:
```
⚠ Current branch is {current}, expected {STORY_BRANCH}.
Either run /story-branch first, or check out the correct branch manually.
```
→ HALT.

### 2b. Story is done in sprint-status.yaml

Read `docs/sprint-status.yaml`. Find the entry matching `currentStory.storyId`. Its `status` must be `done`.

If `in-progress` or anything else:
```
⚠ Story {storyId} is not marked done in sprint-status.yaml.
Pass Gate 5 in /implement before running /story-pr.
```
→ HALT.

### 2c. There are commits to PR

```bash
git log main..HEAD --oneline
```
If empty:
```
⚠ No commits on {STORY_BRANCH} vs main. Nothing to PR.
```
→ HALT.

### 2d. `gh` CLI is authenticated

```bash
gh auth status
```
If it errors out:
```
⚠ gh CLI is not authenticated. Run `gh auth login` and retry.
```
→ HALT.

## 3. Idempotency guard — check for existing PR

```bash
gh pr list --head {STORY_BRANCH} --json url,state,number -q '.[0]'
```

- **Output is empty string or `null`** → no PR exists. Proceed to step 4 (create).
- **Output has `state: "OPEN"`** → PR already exists. Capture `url` as `PR_URL`, **skip to step 6** (poll).
- **Output has `state: "MERGED"`** → PR already merged. **Skip to step 7** (return to main).
- **Output has `state: "CLOSED"`** (not merged) → HALT:
  ```
  ⚠ A closed-but-not-merged PR exists for {STORY_BRANCH}: {url}
  Reopen it on GitHub, or delete the branch and restart the workflow.
  ```

## 4. Build PR body

Write the body to `/tmp/laim-pr-body-{storyId}.md` using the Write tool:

```markdown
## {storyId}: {title}

**Feature:** {feature}
**Branch:** `{STORY_BRANCH}` → `main`

### Commits

{output of: git log main..HEAD --format="- %s"}

### Acceptance Criteria

{extract ## Acceptance Criteria section from docs/stories/{storyId}-*.md if present;
 if section missing, write: "_See story file: docs/stories/{storyId}-*.md_"}

### Verification

- Gate 5 passed per LaiM /implement.
- Full test suite green, lint clean, per-task atomic commits.

---
_Auto-generated by /story-pr. Review and mark ready when satisfied._
```

The story file path pattern: `docs/stories/{storyId}-*.md` (single file). Read it, locate the `## Acceptance Criteria` heading, and include everything until the next `##` heading.

## 5. Push and create the draft PR

```bash
git push -u origin {STORY_BRANCH}
```

Then:
```bash
gh pr create \
  --draft \
  --base main \
  --head {STORY_BRANCH} \
  --title "{storyId}: {title}" \
  --body-file /tmp/laim-pr-body-{storyId}.md
```

Capture the PR URL from stdout as `PR_URL`.

Display:
```
═══ Draft PR opened ═══
  PR:     {PR_URL}
  Branch: {STORY_BRANCH} → main
  Title:  {storyId}: {title}
```

## 6. Block until merged

Enter a poll loop. Every 30 seconds, run:
```bash
gh pr view {PR_URL} --json state,mergedAt -q '.state'
```

Interpret the result:

- **`MERGED`** → break the loop, proceed to step 7.
- **`OPEN`** → print heartbeat `… still open ({elapsed}s elapsed)` and continue.
- **`CLOSED`** (closed without merge) → HALT:
  ```
  ⚠ PR {PR_URL} was closed without merging.
  Reopen on GitHub and rerun /story-pr, or manually `git checkout main` and skip this PR.
  ```

**Implementation note for Claude:** use the `Monitor` tool with an until-loop for efficient polling — you will be notified when the condition is met without needing to proactively sleep/poll. Example shape:

```bash
until [ "$(gh pr view {PR_URL} --json state -q '.state')" = "MERGED" ]; do
  sleep 30
done
```

The user may interrupt with Ctrl-C or type "pause" at any time. If interrupted:
```
Poll paused. PR still open at {PR_URL}.
Re-run /story-pr when you are ready — the idempotency guard (step 3) will pick up here.
```

## 7. Return to main

After merge (or on finding an already-merged PR in step 3):

```bash
git checkout main
git pull --ff-only origin main
```

If `--ff-only` fails:
```
⚠ main could not fast-forward from origin/main — something diverged.
Resolve manually (e.g. git pull --rebase) before starting the next story.
```
→ HALT.

On success, capture the HEAD SHA:
```bash
git log -1 --format='%h %s'
```

Display:
```
═══ Story {storyId} merged — back on main ═══
  Merge commit: {SHA + subject}
  Ready for next story. Run /start to continue.
```

## 8. Cleanup (optional, not automatic)

The local branch `{STORY_BRANCH}` is retained. If the user wants to delete it:
```bash
git branch -d {STORY_BRANCH}
```
Do NOT run this automatically — the user decides.
