## 1. Principles

### 1.1 — Use shadcn, Don't Rebuild
Every component comes from shadcn (`npx shadcn add`). Don't hand-code buttons, inputs, dialogs, tables. No UI library — you own all the code in your project.

### 1.2 — shadcn = Structure, Figma = Skin
Padding, heights, gaps, radius, spacing come from shadcn defaults. Colors, themes, typography, font weights, icon sizes come from this design system. When Figma shows different padding than shadcn, the designer updates Figma — not you.

### 1.3 — Zero Custom CSS on Components
Use library components with their props (variant, size, etc.). Layout containers use Tailwind utilities: `flex`, `grid`, `gap-*`, `p-*`, `rounded-*`, `bg-muted/*`. Nothing else.

### 1.4 — One File, Any Project
This file works for any project that uses the LimeChain design system. The components are universal. Only the organisms (page compositions) change per project.

## 2. Setup

> **No `@limechain/ui` package.** We use shadcn directly — you own all the code in your project. This design system provides the theme, custom atoms, and the Figma→code mapping. shadcn provides everything else.

### Step 1: Initialize shadcn

```bash
npx shadcn@latest init
```

Choose: TypeScript, Tailwind v4, `src/components/ui`, `@/lib/utils`, CSS variables.

### Step 2: Add shadcn components

Add what your project needs. For a typical DeFi app:

```bash
npx shadcn@latest add button card dialog input label select separator switch table tabs badge avatar popover dropdown-menu slider checkbox radio-group accordion collapsible pagination skeleton scroll-area sheet drawer tooltip progress
```

Or add everything: `npx shadcn@latest add --all`

### Step 3: Paste theme CSS

Copy the full CSS block from [Section 3.7](#37-full-theme-css) into your global stylesheet (`app/globals.css` or `src/index.css`), **after** the Tailwind import. This gives you all 4 themes + shadows + font config.

### Step 4: Copy custom components

From [Section 9](#9-custom-components), copy only what your project needs into `components/ui/`. These are components shadcn doesn't have:

| Component | When you need it |
|-----------|-----------------|
| `IconButton` | Icon-only buttons (close, settings, toolbar) |
| `Tag` | Status chips, filter tags |
| `TokenInput` | Crypto amount inputs (swap, deposit, send) |
| `List / ListItem` | Wallet rows, token lists, settings rows |
| `SegmentedControl` | Buy/Sell toggles, view mode switches |
| `DetailRow / DetailList` | Transaction details, fee breakdowns |
| `Disclosure` | Collapsible "Advanced" sections |
| `CollapsibleCard` | Dashboard cards with expand/collapse |
| `DataTableColumnHeader` | Sortable table columns |
| `NavLink` | Top navigation links with active state |
| `InfoPanel` | Key-value info display |
| `Input` (extended) | **Replaces** shadcn's Input — adds `leftIcon`, `rightIcon`, `prefix`, `suffix`, `size` |
| `Separator` (extended) | **Replaces** shadcn's Separator — adds `label` prop for "or" dividers |

### Step 5: Install icons

```bash
npm install lucide-react
```

All UI icons come from Lucide (see [Section 7](#7-icons)). Import only what each component needs.

### Utility function

All components use `cn()` — shadcn creates this for you during `init`:

```ts
// lib/utils.ts (auto-generated by shadcn)
import { clsx, type ClassValue } from "clsx"
import { twMerge } from "tailwind-merge"
export function cn(...inputs: ClassValue[]) { return twMerge(clsx(inputs)) }
```

### That's it

No package. No dependency. You own everything. Update shadcn components with `npx shadcn@latest add <name> --overwrite`. Update the theme by editing your CSS. Update custom atoms by editing the files.

## 7. Icons

**All UI icons MUST come from [Lucide React](https://lucide.dev).** The Figma design system uses the full Lucide icon set (1,500+ icons). Every icon you see in a Figma design has a matching Lucide component. Do not use other icon libraries.

```bash
npm install lucide-react
```

```tsx
// Import only what you need — Lucide is tree-shakeable
import { Search, ChevronDown, X, Settings } from "lucide-react"
```

**Finding the right icon:** Match the icon name from Figma (e.g., `arrow-up-down`) to the PascalCase Lucide import (`ArrowUpDown`). Browse all icons at [lucide.dev/icons](https://lucide.dev/icons).

### 7.1 Icon Sizes

| Context | Size | Tailwind |
|---------|------|----------|
| Inside buttons (with text) | 16px | `size-4` |
| Standalone icon buttons | 20px | `size-5` |
| Large icon buttons | 24px | `size-6` |
| Inline with text | 14px | `size-3.5` |

### 7.2 Icon Colors

| State | Class |
|-------|-------|
| Default | `text-muted-foreground` |
| Hover | `text-foreground` |
| Active | `text-primary` |
| Disabled | `text-muted-foreground/50` |

### 7.3 Opacity Variants

For opacity variants of any color token, use Tailwind's slash syntax. No additional CSS variables needed:

`bg-muted/50`, `text-primary/80`, `border-border/40`, `bg-destructive/20`, `text-accent/50`

### 7.4 Domain Icons (App-Specific)

Domain icons (tokens, wallets, chains, payment methods, flags, social logos) are **not** Lucide icons — they are brand assets specific to each project. Download them from Figma as SVGs and place in `public/`. Reference as `<img src="/wallets/metamask.svg" />`.

Do not use Lucide for brand/logo icons. Do not use brand icon libraries for UI icons.

const sizeConfig = {

import { Separator as SeparatorPrimitive } from "radix-ui"
import { cn } from "@/lib/utils"

function Separator({ className, orientation = "horizontal", decorative = true, label, ...props }: React.ComponentProps<typeof SeparatorPrimitive.Root> & { label?: string }) {
  if (label && orientation === "horizontal") {
    return (
      <div data-slot="separator" role={decorative ? "none" : "separator"} className={cn("flex items-center gap-4", className)}>
        <div className="bg-border h-px flex-1" />
        <span className="text-muted-foreground text-sm">{label}</span>
        <div className="bg-border h-px flex-1" />
      </div>
    )
  }
  return <SeparatorPrimitive.Root data-slot="separator" decorative={decorative} orientation={orientation} className={cn("bg-border shrink-0 data-[orientation=horizontal]:h-px data-[orientation=horizontal]:w-full data-[orientation=vertical]:h-full data-[orientation=vertical]:w-px", className)} {...props} />
}

export { Separator }

## 12. Rules

1. **Use shadcn, don't rebuild.** If shadcn has it, `npx shadcn add` it. If it's a custom atom in Section 9, copy it. Never hand-code components from scratch.
2. **No packages, no wrappers.** Don't install a UI library. shadcn + this file = everything you need.
3. **shadcn = structure.** Don't override padding, heights, gaps, radius. Those are shadcn defaults.
4. **Figma = skin.** Colors, themes, typography, font weights, icon sizes come from the theme CSS in Section 3.7.
5. **No className on component internals.** Only use className for layout: width, margin, spacing of containers.
6. **Tailwind layout only.** `flex`, `grid`, `gap-*`, `p-*`, `rounded-*`, `bg-*`, `space-y-*` — nothing else.
7. **When Figma conflicts with shadcn:** shadcn wins for structure. Tell the designer.
8. **Domain assets are app-specific.** Token icons, wallet SVGs, chain logos — download from Figma, put in `public/`.
9. **Text alignment defaults to left.** Add `text-left` explicitly when parent containers might center.
10. **This file is the source of truth.** When in doubt, reference this document. Not a package. Not a Storybook. This file.

## 15. Interactions

### Transition Timing
| Purpose | Duration | Easing |
|---------|----------|--------|
| Micro (hover, focus) | 150ms | ease-out |
| Small (expand, slide, badge) | 200ms | ease-in-out |
| Medium (dialog, drawer, alert) | 300ms | ease-in-out |
| Fade (skeleton → content) | 150ms | ease-in |

### Component State Behaviors
- **Button:** hover → bg-primary/90, focus → ring-2, active → scale-[0.98], disabled → opacity-50, loading → spinner + opacity-80
- **TokenInput:** default → border-border, focus-within → border-ring, error → border-destructive
- **Token list item:** hover → bg-accent, focus → bg-accent + ring-2, selected → bg-accent/50 + checkmark, disabled → opacity-50
- **Alert variants:** success (green), error (destructive), warning (yellow), info (muted)
- **PriceImpactBadge:** < 1% secondary, >= 1% default, >= 5% amber, >= 15% destructive
- **QuoteFreshnessIndicator:** green → yellow → red dots based on age, spinner when refreshing, flash highlight on output change

### Swap Flow Animations
- Skeleton pulse → fade out (150ms) → quote fade in (150ms)
- SwapButton text crossfade (150ms) between states
- Success/Error alert slides in (200ms) from top of card content
- Output TokenInput border flashes green on success (200ms)
- Card overlay (bg-background/50) during signing/executing

### Keyboard Navigation
- Tab cycles: amount input → input token → swap direction → output token → quote toggle → swap button
- Enter/Space activates buttons and selects tokens
- Escape closes dialogs/drawers and dismisses errors
- Arrow Down/Up navigates token list

## 17. Accessibility Report

### WCAG 2.1 AA Compliance: PASS (with 1 advisory)

**Color Contrast:** All primary combinations pass 4.5:1 minimum. Muted text on light (4.7:1) passes but is close to threshold.

**Advisory (P2):** Dark mode destructive text (#DC2626 on #09090B) is borderline at 4.5:1. Recommend using #EF4444 in dark mode for 5.3:1 ratio.

**Advisory (P3):** Verify amber/yellow warning colors meet 4.5:1 during implementation.

**Keyboard:** Full keyboard navigation defined. Tab order, Enter/Space activation, Escape to close, Arrow keys in lists.

**ARIA:** aria-label on all interactive elements without visible text. role="status" on loading, role="alert" on errors. AvatarFallback for missing logos.

**Touch:** All tap targets ≥44px on mobile. inputMode="decimal" for numeric keyboard.

**Color independence:** All status indicators use icon + text alongside color. PriceImpactBadge uses text prefix "⚠" at warning thresholds.
